# T-EP1-001-002: Create repository interfaces for Product domain

## Title
Create repository interfaces for Product domain

## Description

Create repository interfaces (IProductRepository, IProductPhotoRepository, ICollectionRepository) that define data access contracts for the Product domain entities.

**Purpose:**
- Define data access contracts following the Repository pattern
- Support CRUD operations for all product-related entities
- Include specialized queries (by SKU, by collection, with photos, etc.)

**Details:**
- IProductRepository: CRUD + FindBySku, GetWithPhotos, GetByCollection
- IProductPhotoRepository: CRUD + GetByProductId, UpdateDisplayOrder
- ICollectionRepository: CRUD + FindByName
- Follow existing repository patterns in the codebase
- Include async operations for all methods

**References:**
- Model C4: `Documentos/modelo-c4.md` (Infrastructure layer - Repository pattern)
- Existing repositories: Check UserRepository, PointOfSaleRepository patterns
- Domain entities: Product, ProductPhoto, Collection

## Acceptance Criteria

- [ ] IProductRepository interface created with all required methods
- [ ] IProductPhotoRepository interface created with photo-specific operations
- [ ] ICollectionRepository interface created for collection management
- [ ] Interfaces follow existing repository patterns in codebase
- [ ] Async methods defined for all operations
- [ ] Specialized query methods included (FindBySku, GetWithPhotos, etc.)
