# T-EP1-001-007: Implement Product service with CRUD operations

## Title
Implement Product service with CRUD operations

## Description

Implement ProductService that provides business logic for product management including CRUD operations, validation, and business rules enforcement.

**Purpose:**
- Centralize product business logic and validation
- Provide clean interface for product operations
- Enforce business rules (SKU uniqueness, price validation, etc.)
- Support both individual and bulk operations

**Details:**
- IProductService interface with CRUD methods
- ProductService implementation with repository dependency
- Business validations: SKU uniqueness, price > 0, required fields
- Support for creating/updating products with collection assignment
- Include audit fields (CreatedAt, UpdatedAt) management
- Proper error handling with domain exceptions
- Support for soft deletes (IsActive flag)

**References:**
- User Stories: HU-EP1-001, HU-EP1-002, HU-EP1-003 (CRUD operations)
- Domain entities: Product entity with business rules
- Existing services: Check UserService, PointOfSaleService patterns

## Acceptance Criteria

- [ ] IProductService interface defined with all CRUD operations
- [ ] ProductService implements business logic and validations
- [ ] SKU uniqueness validation across all products
- [ ] Price validation (must be > 0)
- [ ] Collection assignment validation
- [ ] Proper error handling with domain exceptions
- [ ] Audit fields automatically managed
- [ ] Soft delete support with IsActive flag

## Priority
High (foundation for all product operations)
