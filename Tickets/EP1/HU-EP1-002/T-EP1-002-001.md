# T-EP1-002-001: Create product creation API endpoint

## Title
Create product creation API endpoint

## Description

Implement POST /api/products endpoint that allows administrators to create individual products with proper validation and business rule enforcement.

**Purpose:**
- Provide REST API for manual product creation
- Validate input data and enforce business rules
- Create products with proper audit trails
- Return created product information

**Details:**
- POST /api/products endpoint with JSON payload
- Validate required fields (SKU, Name, Price)
- Enforce business rules (unique SKU, price > 0)
- Support optional collection assignment
- Set IsActive = true by default
- Return created product with generated ID
- Proper authentication (Administrator role required)

**References:**
- User Story: `Documentos/Historias/HU-EP1-002.md` (all scenarios)
- Existing endpoints: Check UsersController POST patterns
- Authentication: Use [Authorize(Roles = "Administrator")]
- Product DTOs: Create CreateProductRequest, ProductResponse DTOs

## Acceptance Criteria

- [ ] POST /api/products endpoint implemented
- [ ] Input validation for required fields (SKU, Name, Price)
- [ ] SKU uniqueness validation
- [ ] Price validation (must be > 0)
- [ ] Optional collection assignment
- [ ] Product created with IsActive = true
- [ ] Proper response with created product data
- [ ] Authentication required (Administrator role)
- [ ] API documentation with Swagger
