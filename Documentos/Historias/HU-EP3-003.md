# HU-EP3-003: Registrar venta con precio manual modificado

## Formato estándar

Como **operador**, quiero **poder modificar el precio de venta de un producto al registrar una venta** para **aplicar ajustes de precio autorizados por el negocio en el punto de venta**.

---

## Descripción

Cuando un punto de venta tiene habilitada la opción `AllowManualPriceEdit`, el formulario de registro de venta (tanto manual como con reconocimiento de imagen) debe mostrar un campo editable de precio. El precio oficial del producto se muestra como referencia y se precarga como valor por defecto. Si el operador modifica el precio, el sistema registra la venta con el precio modificado y almacena metadatos de auditoría (`PriceWasOverridden`, `OriginalProductPrice`). Si el POS no permite edición de precio, el sistema rechaza cualquier intento de enviar un precio manual.

**Contexto de negocio:**
- Los operadores en ciertos puntos de venta necesitan flexibilidad para ajustar precios (descuentos, negociaciones)
- Se requiere trazabilidad completa de los precios modificados para auditoría
- El precio oficial del producto siempre se conserva como referencia
- El historial de ventas y el detalle deben mostrar claramente cuándo un precio fue modificado

**Reglas de resolución de precio:**
1. Si el POS no permite edición manual: usar siempre el precio oficial del producto
2. Si el POS permite edición y no se envía precio: usar el precio oficial del producto
3. Si el POS permite edición y se envía un precio igual al oficial: no marcar como modificado
4. Si el POS permite edición y se envía un precio diferente al oficial: marcar como modificado y guardar el precio oficial como referencia

**Campos afectados en `Sale`:**
- `PriceWasOverridden` (boolean, default `false`)
- `OriginalProductPrice` (decimal nullable, solo cuando hay override)

---

## Criterios de Aceptación

### Escenario 1: Venta con precio modificado exitosamente
**Dado que** soy un operador asignado a un punto de venta con `AllowManualPriceEdit = true`
**Y** selecciono un producto con precio oficial de €100
**Cuando** modifico el precio a €75 y confirmo la venta
**Entonces** el sistema registra la venta con `Sale.Price = 75`
**Y** `PriceWasOverridden = true`
**Y** `OriginalProductPrice = 100`
**Y** el stock se actualiza normalmente

### Escenario 2: Venta sin modificar precio cuando está permitido
**Dado que** soy un operador asignado a un punto de venta con `AllowManualPriceEdit = true`
**Y** selecciono un producto con precio oficial de €100
**Cuando** no modifico el precio y confirmo la venta
**Entonces** el sistema registra la venta con `Sale.Price = 100`
**Y** `PriceWasOverridden = false`
**Y** `OriginalProductPrice = null`

### Escenario 3: Rechazo de precio manual cuando POS no lo permite
**Dado que** soy un operador asignado a un punto de venta con `AllowManualPriceEdit = false`
**Cuando** envío una solicitud de venta con un precio manual
**Entonces** el sistema rechaza la venta con error 400
**Y** muestra mensaje: "Manual price editing is not allowed for this point of sale"
**Y** no se crea registro de venta

### Escenario 4: Validación de precio positivo
**Dado que** el punto de venta permite edición de precio
**Cuando** intento registrar una venta con precio <= 0
**Entonces** el sistema rechaza la venta con error de validación
**Y** muestra mensaje: "Price must be greater than zero"

### Escenario 5: Campo de precio oculto cuando POS no permite edición
**Dado que** soy un operador asignado a un punto de venta con `AllowManualPriceEdit = false`
**Cuando** abro el formulario de registro de venta
**Entonces** no se muestra el campo de precio manual
**Y** el precio del producto se usa directamente sin posibilidad de edición

### Escenario 6: Precio prefillado con valor oficial
**Dado que** el punto de venta permite edición de precio
**Y** selecciono un producto con precio oficial de €100
**Cuando** el formulario muestra el campo de precio manual
**Entonces** el campo tiene como placeholder el precio oficial (€100)
**Y** si no modifico el valor, se usa el precio oficial

### Escenario 7: Indicador "Precio modificado" en historial de ventas
**Dado que** existen ventas con `PriceWasOverridden = true`
**Cuando** consulto el historial de ventas
**Entonces** las ventas con precio modificado muestran un indicador visual "Precio modificado"
**Y** el detalle de la venta muestra el precio de venta y el precio oficial original

### Escenario 8: Confirmación visual antes de enviar precio modificado
**Dado que** el punto de venta permite edición de precio
**Y** he modificado el precio del producto
**Cuando** abro el diálogo de confirmación de venta
**Entonces** muestra el precio modificado
**Y** muestra el precio oficial como referencia con indicación de "(modificado)"
**Y** permite confirmar o cancelar la venta
