# T-EP1-001-008: Create product import API endpoint

## Title
Create product import API endpoint

## Description

Implement POST /api/products/import endpoint that accepts Excel files and triggers bulk product import operations with proper authentication and validation.

**Purpose:**
- Provide REST API endpoint for Excel import functionality
- Handle file uploads securely with size and type validation
- Integrate with ExcelImportService for business logic
- Return detailed import results to the client

**Details:**
- POST /api/products/import endpoint with file upload
- Accept multipart/form-data with Excel file
- File validation: size limit (10MB), format (.xlsx/.xls), content type
- Use ExcelImportService for processing logic
- Return ImportResult with success/error details
- Proper authentication (Administrator role required)
- Error handling for invalid files, processing errors, etc.

**References:**
- User Story: `Documentos/Historias/HU-EP1-001.md` (all import scenarios)
- Existing controllers: Check AuthController, UsersController patterns
- Authentication: Use [Authorize(Roles = "Administrator")] like other admin endpoints
- File upload guide: `Documentos/Testing/Backend/09-archivos-uploads.md`

## Acceptance Criteria

- [ ] POST /api/products/import endpoint implemented
- [ ] File upload validation (size, format, content type)
- [ ] Authentication required (Administrator role)
- [ ] Integration with ExcelImportService
- [ ] Detailed response with import results (success count, errors)
- [ ] Proper error handling for invalid files
- [ ] API documentation with Swagger annotations
