# T-EP1-001-003: Create initial database migration for Product entities

## Title
Create initial database migration for Product entities

## Description

Create Entity Framework Core migration to add Product, ProductPhoto, and Collection tables to the database according to the data model specification.

**Purpose:**
- Set up database schema for product management
- Create tables with proper constraints, indexes, and relationships
- Enable data persistence for product operations

**Details:**
- Product table: Id (PK), SKU (unique index), Name, Description, Price, CollectionId (FK), IsActive, CreatedAt, UpdatedAt
- ProductPhoto table: Id (PK), ProductId (FK), FileName, DisplayOrder, IsPrimary, CreatedAt, UpdatedAt
- Collection table: Id (PK), Name, Description, CreatedAt, UpdatedAt
- Foreign key relationships and constraints
- Indexes on frequently queried columns (SKU, CollectionId)
- Follow existing migration patterns in codebase

**References:**
- Data Model: `Documentos/modelo-de-datos.md` (table specifications)
- Existing migrations: Check migration files in Infrastructure/Data/Migrations/
- Entity configurations: Follow UserConfiguration, PointOfSaleConfiguration patterns

## Acceptance Criteria

- [ ] Migration file created with Add-Migration command
- [ ] Product table created with all required columns and constraints
- [ ] ProductPhoto table created with proper relationships
- [ ] Collection table created for product categorization
- [ ] Foreign key constraints and indexes properly defined
- [ ] Migration can be applied successfully to database
- [ ] Database schema matches data model specification
