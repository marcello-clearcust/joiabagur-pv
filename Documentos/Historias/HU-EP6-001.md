# HU-EP6-001: Configurar métodos de pago disponibles en el sistema

## Formato estándar

Como **administrador**, quiero **configurar los métodos de pago disponibles en el sistema** para **definir qué métodos pueden ser asignados a los puntos de venta**.

---

## Descripción

El sistema debe permitir a los administradores gestionar la lista general de métodos de pago disponibles. En el MVP, estos métodos están predefinidos pero deben poder activarse/desactivarse. Los métodos de pago predefinidos son: Efectivo (CASH), Bizum (BIZUM), Transferencia bancaria (TRANSFER), Tarjeta TPV propio (CARD_OWN), Tarjeta TPV punto de venta (CARD_POS), PayPal (PAYPAL).

**Contexto de negocio:**
- Los métodos de pago deben estar disponibles en el sistema antes de asignarlos a puntos de venta
- Los métodos pueden activarse/desactivarse según necesidades del negocio
- El código del método debe ser único e inmutable

---

## Criterios de Aceptación

### Escenario 1: Visualizar métodos de pago predefinidos
**Dado que** soy un administrador autenticado  
**Cuando** accedo a la gestión de métodos de pago  
**Entonces** veo la lista de métodos de pago predefinidos con su código, nombre y descripción  
**Y** puedo ver su estado (activo/inactivo)

### Escenario 2: Activar método de pago
**Dado que** existe un método de pago inactivo  
**Cuando** lo activo  
**Entonces** `IsActive = true`  
**Y** el método está disponible para asignar a puntos de venta

### Escenario 3: Desactivar método de pago
**Dado que** existe un método de pago activo  
**Cuando** lo desactivo  
**Entonces** `IsActive = false`  
**Y** el método no puede asignarse a nuevos puntos de venta  
**Y** los puntos de venta que ya lo tienen asignado pueden seguir usándolo (o mostrar advertencia según regla de negocio)

### Escenario 4: Editar nombre y descripción de método de pago
**Dado que** existe un método de pago  
**Cuando** edito su nombre o descripción  
**Entonces** el sistema actualiza la información exitosamente  
**Y** el código permanece inmutable

---

## Notas adicionales

- **Métodos predefinidos:** Se crean mediante seed/migración inicial
- **Código inmutable:** El código (CASH, BIZUM, etc.) no debe poder modificarse
- **Validaciones:** Código único, nombre requerido

---

## Tareas

1. Backend: Seed de métodos de pago predefinidos
2. Backend: Endpoints `GET /api/payment-methods`, `PUT /api/payment-methods/{id}`
3. Frontend: Lista y edición de métodos de pago
4. Validaciones: Código inmutable, solo administradores
