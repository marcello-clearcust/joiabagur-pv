# T-EP1-003-001: Create product update API endpoint

## Title
Create product update API endpoint

## Description

Implement PUT /api/products/{id} endpoint for updating existing products with proper validation and business rules, ensuring SKU immutability.

**Purpose:**
- Allow administrators to update product information
- Maintain SKU immutability as business rule
- Update audit fields and timestamps
- Validate input and enforce business rules

**Details:**
- PUT /api/products/{id} endpoint with JSON payload
- Allow updates to: Name, Description, Price, CollectionId
- Prevent SKU updates (return error if attempted)
- Validate price > 0 and other business rules
- Update UpdatedAt timestamp
- Return updated product information
- Proper authentication (Administrator role)

**References:**
- User Story: `Documentos/Historias/HU-EP1-003.md` (all scenarios)
- Existing endpoints: Check UsersController PUT patterns
- Product service: Reuse ProductService update logic

## Acceptance Criteria

- [ ] PUT /api/products/{id} endpoint implemented
- [ ] Updates allowed for Name, Description, Price, Collection
- [ ] SKU updates rejected with error message
- [ ] Price validation enforced
- [ ] UpdatedAt timestamp updated
- [ ] Proper response with updated product
- [ ] Authentication required (Administrator role)
