# HU-EP10-005: Advertencia de desviación de precio (10%)

## Formato estándar

Como **administrador**, quiero **recibir una advertencia cuando el precio oficial del producto difiere más del 10% del precio sugerido por componentes** para **mantener consistencia entre el precio de venta y el coste real**.

---

## Descripción

Cuando el producto tiene al menos un componente asignado y `|Product.Price - TotalSalePrice| / TotalSalePrice > 0.10`, se muestra una advertencia en la pantalla de edición. Incluye un botón de acción rápida "Ajustar a precio sugerido" que actualiza `Product.Price` al valor de `TotalSalePrice` calculado.

---

## Requisitos funcionales

- RF-001: Calcular desviación: `|Product.Price - TotalSalePrice| / TotalSalePrice`.
- RF-002: Mostrar advertencia solo cuando el producto tiene al menos un componente asignado y la desviación es > 10%.
- RF-003: Mensaje de advertencia: "El precio oficial (€X) difiere más del 10% del precio sugerido por componentes (€Y). ¿Desea ajustar?"
- RF-004: Botón "Ajustar a precio sugerido" que asigna `Product.Price = TotalSalePrice`.
- RF-005: Tras ajustar, la advertencia desaparece (o se actualiza si aún hay desviación por redondeo).
- RF-006: Si TotalSalePrice es 0 (ej. todos los componentes con SalePrice 0), no mostrar advertencia para evitar división por cero.

---

## Requisitos no funcionales

- RNF-001: Advertencia visible solo en pantalla de edición de producto.
- RNF-002: Solo administradores ven la advertencia y el botón.
- RNF-003: Cálculo en tiempo real al cambiar Product.Price o componentes.

---

## Criterios de Aceptación

### Escenario 1: Advertencia cuando desviación > 10%
**Dado que** el producto tiene TotalSalePrice = 100  
**Y** Product.Price = 120  
**Cuando** visualizo la pantalla de edición  
**Entonces** aparece advertencia: el precio oficial (€120) difiere más del 10% del sugerido (€100)  
**Y** se muestra el botón "Ajustar a precio sugerido"

### Escenario 2: Sin advertencia cuando desviación <= 10%
**Dado que** el producto tiene TotalSalePrice = 100  
**Y** Product.Price = 108  
**Cuando** visualizo la pantalla de edición  
**Entonces** no aparece advertencia  
**Y** el botón de ajuste no se muestra

### Escenario 3: Acción rápida ajustar precio
**Dado que** aparece la advertencia con Product.Price = 120 y TotalSalePrice = 100  
**Cuando** pulso "Ajustar a precio sugerido"  
**Entonces** Product.Price se actualiza a 100  
**Y** la advertencia desaparece  
**Y** el formulario refleja el nuevo precio (debe guardarse para persistir)

### Escenario 4: Sin componentes
**Dado que** el producto no tiene componentes asignados  
**Cuando** visualizo la pantalla de edición  
**Entonces** no se muestra advertencia de desviación  
**Y** no se calcula TotalSalePrice para este propósito

---

## Tareas

1. Frontend: Cálculo de desviación en tiempo real
2. Frontend: Componente de advertencia (Alert) con mensaje y botón
3. Frontend: Lógica del botón "Ajustar a precio sugerido" que actualiza el campo Price del formulario
4. Validación: evitar división por cero cuando TotalSalePrice = 0
