# T-EP1-001-001: Create Product and Collection domain entities

## Title
Create Product and Collection domain entities

## Description

Create the core domain entities Product, ProductPhoto, and Collection according to the data model specification. These entities represent the fundamental business objects for product management.

**Purpose:**
- Define the domain model for products, product photos, and collections
- Establish relationships between entities (Product belongs to Collection, Product has multiple ProductPhotos)
- Include all required properties and business rules

**Details:**
- Product: SKU (unique), Name, Description, Price, CollectionId (optional), IsActive, timestamps
- ProductPhoto: ProductId, FileName, DisplayOrder, IsPrimary, timestamps
- Collection: Name, Description, timestamps
- Implement proper domain validations (price > 0, SKU uniqueness, etc.)
- Follow the existing BaseEntity pattern in the codebase

**References:**
- Model C4: `Documentos/modelo-c4.md` (Level 3: Components - Domain Layer)
- Data Model: `Documentos/modelo-de-datos.md` (Product, ProductPhoto, Collection entities)
- Architecture: `Documentos/arquitectura.md` (Domain layer patterns)

## Acceptance Criteria

- [ ] Product entity created with all required properties and validations
- [ ] ProductPhoto entity created with display order and primary photo logic
- [ ] Collection entity created for product grouping
- [ ] Proper relationships established between entities
- [ ] Domain validations implemented (price validation, SKU format)
- [ ] Entities follow existing BaseEntity pattern
- [ ] Unit tests cover entity validations and business rules
