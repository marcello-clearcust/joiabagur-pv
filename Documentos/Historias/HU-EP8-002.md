# HU-EP8-002: Editar punto de venta existente

## Formato estándar

Como **administrador**, quiero **editar información de puntos de venta existentes** para **actualizar sus datos cuando cambian ubicación o información de contacto**.

---

## Descripción

El sistema debe permitir a los administradores modificar la información de puntos de venta existentes, incluyendo nombre, dirección, teléfono, email y estado activo/inactivo. El código debe ser inmutable para mantener integridad histórica.

**Contexto de negocio:**
- Los puntos de venta pueden cambiar de ubicación o información de contacto
- Puede ser necesario desactivar puntos de venta temporalmente
- El código debe permanecer constante para mantener trazabilidad histórica

---

## Criterios de Aceptación

### Escenario 1: Editar información básica exitosamente
**Dado que** soy un administrador autenticado  
**Y** existe un punto de venta en el sistema  
**Cuando** edito su nombre, dirección, teléfono o email  
**Entonces** el sistema actualiza el punto de venta exitosamente  
**Y** se actualiza el campo `UpdatedAt`  
**Y** se muestra mensaje de confirmación

### Escenario 2: Código inmutable
**Dado que** estoy editando un punto de venta  
**Cuando** intento cambiar el código  
**Entonces** el campo código está deshabilitado o no está presente  
**Y** el sistema no permite cambiar el código

### Escenario 3: Desactivar punto de venta
**Dado que** existe un punto de venta activo  
**Cuando** cambio su estado a inactivo (`IsActive = false`)  
**Entonces** el sistema actualiza el estado exitosamente  
**Y** el punto de venta no aparece en listas para nuevas operaciones  
**Y** se mantiene el historial de ventas e inventario

---

## Tareas

1. Backend: Endpoint `PUT /api/point-of-sales/{id}`
2. Frontend: Formulario de edición pre-poblado
3. Validaciones: Código inmutable, email válido si se proporciona
